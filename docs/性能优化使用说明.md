# AI助手性能优化使用说明

## 📋 配置文件说明

### 主要配置文件
- `config/ai_performance_config_commented.json` - **实际使用的配置文件**（带详细注释）
- `config/ai_performance_config.json` - 备用配置文件

### 🎯 直接使用带注释的配置文件
系统现在默认使用 `ai_performance_config_commented.json` 文件，您可以直接修改这个文件来调整性能设置。所有参数都有详细的中文注释说明。

## 🔧 参数调整指南

### 如果响应太慢：
1. 开启 `enable_fast_mode: true`
2. 减少 `max_tokens` 到 1000-1500
3. 增加 `similarity_threshold` 到 0.4-0.5
4. 减少 `vector_search_k` 到 2

### 如果想要AI生成图表：
1. 设置 `enable_mermaid_generation: true`
2. 增加 `temperature` 到 0.4-0.7（鼓励创造性）
3. 增加 `max_tokens` 到 4000-5000（图表需要更多空间）
4. 增加 `timeout_seconds` 到 45-60（图表生成需要更长时间）

### 如果回答不够准确：
1. 关闭 `enable_fast_mode: false`
2. 增加 `max_tokens` 到 2500-3000
3. 减少 `similarity_threshold` 到 0.2-0.3
4. 增加 `vector_search_k` 到 4-5

### 如果内存占用太高：
1. 减少 `max_cache_size` 到 20-30
2. 增加 `stream_chunk_size` 到 20-50

### 如果流式显示不够流畅：
1. 减少 `stream_chunk_size` 到 3-5（字符级别分块）
2. 如果显示太频繁，可以增加到 10-20

### 流式分块工作原理：
- **真正的流式分块**：在API流式响应层面进行分块，而不是收到完整响应后分块
- **文本缓冲区**：累积接收到的文本片段，按配置的块大小输出
- **实时响应**：AI回答会按照配置的字符数实时显示，提供更好的用户体验

## 💡 使用技巧

1. **首次使用**：建议先用方案2（平衡性能）
2. **根据需求调整**：如果觉得慢就切换到方案1，如果觉得不够准确就切换到方案3
3. **观察监控**：通过性能监控了解实际响应时间
4. **定期清理**：如果缓存太多，可以重启程序清理缓存

## 🚀 预期效果

- **首次提问**：2-3秒（正常）
- **重复提问**：0.5-1秒（利用缓存）
- **相似问题**：1-2秒（智能匹配）

## ❓ 常见问题

**Q: 修改配置后没有效果？**
A: 需要重启AI助手程序才能生效

**Q: 缓存占用太多内存？**
A: 减少 `max_cache_size` 参数

**Q: 网络不稳定经常超时？**
A: 增加 `timeout_seconds` 参数

**Q: 想要更快的响应？**
A: 开启 `enable_fast_mode` 并减少 `max_tokens`
